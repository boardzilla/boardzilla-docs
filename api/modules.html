createGame​

▸ createGame<P, B>(options): SetupFunction<P, B>

Create your game

Type parameters​

NameType
Pextends Player
Bextends Board<P>

Parameters​

NameTypeDescription
optionsObjectAll the options needed to define your game are listed below.
options.actions(board: B, actionFunction: typeof action, player: P) => Record<string, Action<P, Argument<P>[]>>A function that provides an object defining all the actions in your game. The function accepts 3 arguments: - an instance of the boardClass above - the action function used to define each action. - the player taking the action, an instance of playerClass above Each key is a unique action name and the value is the result of calling action.
options.boardClassElementClass<P, B>Your board class. This must extend Board. If you do not need any custom Board attributes or behaviour, simply put Board here. This becomes the B type generic used here.
options.breakpoints?(aspectRatio: number) => stringA function that determines which layout breakpoint to use. The function accepts the aspect ratio of the current player's viewable area and returns the name of the breakpoint.
options.elementClasses?ElementClass<P, GameElement<P>>[]An array of all other board classes you declare that will be used in your board. These all must extend Space or Piece.
options.flow(board: B) => default<P>A function that defines your game's flow. This function accepts a single argument which is an instance of the boardClass above. The function should return the result of one of the flow functions: - whileLoop - forEach - forLoop - eachPlayer - ifElse - switchCase - playerActions
options.layout?(board: B, breakpoint: string) => voidA function that defines all the layout rules for your game. See GameElement#layout, GameElement#appearance and Board#layoutStep. Function accepts two arguments: - an instance of the boardClass above - the breakpoint string from your breakpoints function, or '_default' if none specified.
options.playerClass(a: PlayerAttributes<P>) => PYour player class. This must extend Player. If you do not need any custom Player attributes or behaviour, simply put Player here. This becomes the P type generic used throughout boardzilla.
options.setup?(board: B) => anyA function that sets up the game. This function accepts a single argument which is an instance of the boardClass above. The function should add all the spaces and pieces you need before your game can start.
